Evolve AT Health Portal from an internal intranet into a scalable multi-tenant pharmacy operations platform, with AT Health as the first organisation.

Do not rewrite the whole app unless necessary. Extend the current architecture so it is suitable for internal use now and future resale to other pharmacy groups.

CORE PRODUCT DIRECTION
Build this as a multi-tenant pharmacy operations platform where:
- one platform can host multiple organisations
- each organisation can have multiple pharmacies
- each pharmacy can have users, settings, roles, dashboards, reports, documents, templates, and permissions
- AT Health is the first tenant/organisation

1) MULTI-TENANT FOUNDATION
Add data structure and permissions for:
- Platform
- Organisation
- Pharmacy
- User
- Staff identity
- Role
- Module access
- Audit logs per organisation
Ensure strict tenant separation so all records are organisation-scoped and queries are filtered correctly.

2) PLATFORM ADMIN VS ORGANISATION ADMIN
Create two admin levels:
- Platform Admin
  - manages organisations, packages, feature toggles, global templates, white-labelling, subscription readiness
- Organisation Admin
  - manages their own pharmacies, users, templates, settings, reports, targets, and workflows

AT Health should operate as an organisation tenant, while the platform supports future additional pharmacy groups.

6) ACCESS CONTROL AND LOGIN ARCHITECTURE
Keep the strict two-layer access model:
Layer 1 = email account access
Layer 2 = person identity by PIN

Enforce current email-to-scope mapping for AT Health:
- ahmed@at-health.co.uk = Ahmed / Master only
- info@at-health.co.uk = Head Office Admin only
- finance@at-health.co.uk = shared Finance login with finance staff picker
- pharmacy emails are branch-only

Requirements:
- Ahmed login shows Ahmed only
- Head Office email only shows Head Office staff identities
- Finance only shows Finance identities
- Pharmacy emails only show that branch’s staff identities
- No registration or public sign-up
- Separate branch session and user session
- Log out of User = drops PIN identity only
- Log out of Branch = drops full email session
- OTP via mock/live auth mode
- Microsoft 365 SMTP ready
- Auth Mode toggle in Launch Control:
  - Mock
  - SMTP Live

Add dashboard comparison metrics:
